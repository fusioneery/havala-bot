## 1. Публичные группы (user-managed groups)

**Статус:** Planned / Discussable

Сейчас бот мониторит только доверенные группы, указанные администратором. Цель — дать пользователям возможность подключать свои группы.

### Как это работает

1. Пользователь добавляет бота в Telegram-группу как админа
2. Бот регистрирует группу как "публичную" и она появляется в общем каталоге
3. Другие пользователи могут подписаться на публичную группу в настройках, чтобы офферы из неё учитывались в их поиске

### Типы групп

| Тип | Описание |
|-----|----------|
| **Trusted** | Доверенные группы, проверенные оператором |
| **Public** | Группы, добавленные пользователями |

### Лимиты

Администратор может ограничить:
- Максимальное количество публичных групп в системе
- Максимальное количество публичных групп, на которые может подписаться один пользователь

### UI

- Раздел "Группы" в настройках
- Каталог публичных групп с поиском
- Переключатели для включения/выключения групп в свой фид
- Индикаторы: количество участников, активность, дата добавления

### Риски

- Быстрое захламление фейковыми или просто мусорными группами и офферами.

## 2. Система репутации

**Статус:** Planned / Discussable

Репутация помогает пользователям оценить надёжность контрагента до начала обмена.

### Концепция

Репутация рассчитывается на основе:
- **Количества успешных обменов** с конкретным контрагентом
- **Репутации контрагентов**, с которыми человек менялся (рекурсивно)
- **Расстояния в социальном графе** — чем ближе к тебе человек (друг vs друг друга), тем выше вес

### Открытые вопросы

- Как избежать накрутки репутации через фейковые обмены и добавления в друзья фейков друг друга?
- Нужен ли decay (уменьшение репутации со временем)?
- Как обрабатывать негативные исходы (споры, отмены)?
- Как отображать репутацию: число, звёзды, уровни?
- Должна ли репутация быть публичной или видна только в контексте матча?

### UI

- Отображение репутации в карточке матча
- Подробная разбивка по клику (кол-во обменов, граф связей)

### Риски

- Вечная гонка между абьюзерами и админами.
Доверенная группа не просто так считается доверенной - она должна хорошо фильтроваться админами. Если мы добавляем больше юзеров из внешних публичных групп, то мы перекладываем головняк на механизм репутации и самих пользователей. Основная ценность продукта всё же в закрытом круге друзей и знакомых. Не хотим переизобретать P2P площадку в очередной раз.

## 3. Админ-панель

**Статус:** Planned

Веб-интерфейс для администраторов бота с возможностью просмотра статистики и модерации.

### Статистика

- Общее количество пользователей, активных за период
- Количество офферов (всего / активных / завершённых)
- Количество матчей за период
- Популярные валютные пары
- Активность по группам

### Модерация

- Список пользователей с поиском и фильтрами
- Возможность забанить/разбанить пользователя
- Просмотр публичных групп, возможность удалить группу
- Логи действий модерации

### Настройки

- Управление доверенными группами через UI
- Настройка лимитов без редеплоя

### Авторизация

Вход через Telegram для пользователей из списка админов.

## 4. Онбординг

**Статус:** Planned

Серия экранов при первом запуске, объясняющих принципы работы сервиса.

### Экраны

1. **Welcome** — что это за бот и зачем он нужен
2. **Как это работает** — схема: создаёшь оффер → бот ищет → получаешь матч
3. **Доверие** — объяснение системы видимости (друзья / знакомые знакомых)
4. **Безопасность** — бот не касается денег, вы договариваетесь напрямую
5. **Готово** — CTA создать первый оффер

### Принципы

- Можно пропустить в любой момент
- Можно вернуться из настроек ("Показать онбординг заново")
- Анимации для вовлечения, но не навязчивые
- Короткие тексты, визуальные схемы
